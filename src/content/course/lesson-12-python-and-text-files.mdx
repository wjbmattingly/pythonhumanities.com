For now, the point is simple: text files are an essential building block of any DH project. You will use them in your own work and you will encounter them in everyone else's. Understanding how to open them, read from them, write to them, and rip the data out of them is therefore foundational.

## What is a text file?

A **text file** is a file with the extension `.txt` containing plain, human-readable characters. In DH projects we use text files to store things like full-text manuscripts, lists of names separated by lines, transcripts, or letters. If you separate fields on each line with commas (or tabs), the same `.txt` file can be read as a CSV. For now, we'll just treat text files as one big string per file.

## Opening a file with `with`

In Python, the modern way to open a file is the `with` operator. **The `with` block automatically closes the file when you leave it**, which means you cannot accidentally leave files open in memory — a common bug in scripts that loop over many files.

Here is the canonical pattern: create a file called `text.txt`, write `"Hello"` to it, then read it back line by line.

```python
file = "text.txt"

with open(file, "a+", encoding="utf-8") as f:
    f.write("Hello")

with open(file, "r", encoding="utf-8") as f:
    for line in f.readlines():
        print(line)
```

A few things to note:

- `file` is a string — the name (or path) of the file. If it lived in a subfolder, you'd write something like `"data/text.txt"`.
- `open(file, "a+", encoding="utf-8")` opens the file in **append-and-read** mode and creates it if it doesn't exist. The string `"a+"` is the **file handle** (more on these below).
- Inside the `with` block, the file is bound to the name `f`. The convention is `f` (sometimes `fp` for "file pointer").
- `f.write("Hello")` writes the string. Notice it does *not* add a newline; you'd write `"Hello\n"` to do that.
- The second `with` block opens the file again, this time in read mode (`"r"`), and iterates the lines.
- Always set `encoding="utf-8"` when working with text. The default differs across operating systems, and UTF-8 is the modern standard.

Try editing the example to use string functions you learned in [Lesson 03](/course/lesson-03-python-strings/) — strip whitespace from each line, uppercase it, count occurrences of a substring.

## File handles

The second argument to `open` controls **how** Python opens the file. The handles you should know:

- `r` — read only (the default).
- `r+` — read and write.
- `w` — write only; **truncates** the file if it exists.
- `w+` — write and read; also truncates.
- `a` — append only; creates the file if it doesn't exist.
- `a+` — append and read; creates the file if it doesn't exist.

The big distinction is between `w*` (which destroys existing content) and `a*` (which preserves it). When in doubt, prefer `a+` while you're learning so you don't accidentally wipe out a file.
