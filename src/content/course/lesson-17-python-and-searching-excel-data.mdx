Reading a spreadsheet is one thing; finding the rows you actually care about is the work. This lesson turns the file-loading from Lesson 16 into the foundation for any tabular DH project: walk the rows, test each one, collect the matches.

We'll use `openpyxl` here. (The same pattern works with `xlrd` for `.xls` files â€” the only difference is the API for getting at cell values.)

## Step 1 â€” load the data into a list of dictionaries

A spreadsheet is just a list of records. The most useful shape for working with one in Python is a list of dictionaries â€” one dictionary per row, keyed by the header.

Suppose the workbook has columns `sender`, `recipient`, `year`, `place`:

```python
from openpyxl import load_workbook

wb = load_workbook("letters.xlsx", data_only=True)
sheet = wb.active

rows = list(sheet.iter_rows(values_only=True))
header, *body = rows

records = [dict(zip(header, r)) for r in body]
print(records[0])
# {'sender': 'Voltaire', 'recipient': 'Ã‰milie', 'year': 1735, 'place': 'Cirey'}
```

Once your data is in this shape, the rest of the lesson is just list-of-dictionaries work â€” no spreadsheet code beyond this point.

## Exact match

```python
target = "Paris"
matches = [r for r in records if r["place"] == target]
```

That's a list comprehension (Lesson 06): for each row in `records`, keep it if its `place` field equals `target`. The result is a list of full row-dictionaries â€” same shape as `records` itself, just shorter.

## Case-insensitive partial match

Real searches are rarely exact. Lowercase both sides and use `in`:

```python
needle = "paris"
matches = [
    r for r in records
    if needle in str(r["place"]).lower()
]
```

`str(...)` is a guard against `None` values (empty cells) and numeric cells that don't have a `.lower()` method. Always wrap a cell value in `str` before doing string operations on it.

## Multiple criteria

Combine conditions with `and`/`or`. Letters from Voltaire after 1750:

```python
matches = [
    r for r in records
    if r["sender"] == "Voltaire" and r["year"] and r["year"] > 1750
]
```

The middle `r["year"]` check handles rows where `year` is missing â€” `None > 1750` would raise a `TypeError`, but `None and â€¦` short-circuits before we get there.

## Counting and aggregating

`Counter` (from Lesson 13) turns a list of values into a frequency table:

```python
from collections import Counter

senders = Counter(r["sender"] for r in records if r["sender"])
print(senders.most_common(5))
# [('Voltaire', 8421), ('Ã‰milie', 192), ...]

by_year = Counter(r["year"] for r in records if r["year"])
print(sorted(by_year.items())[:10])
```

For aggregations beyond counts (sums, means, group-bys with multiple stats), this is the point at which **pandas** or **polars** start to pull ahead. They aren't in this course, but if you ever find yourself building a `defaultdict(list)` to group records by some key, that's the moment to consider switching.

## Sorting

`sorted` with a `key` function (Lesson 06) does the heavy lifting:

```python
top_senders = sorted(records, key=lambda r: r["sender"] or "")
top_letters = sorted(
    [r for r in records if r["year"]],
    key=lambda r: r["year"],
    reverse=True,
)[:10]
```

`lambda r: r["year"]` is a one-line anonymous function â€” equivalent to writing a named function that returns `r["year"]`. Use lambdas when the function is throwaway and tiny. For anything more complex, `def` it.

## Writing the matches back out

Once you have the filtered list, you can write it to a new spreadsheet (Lesson 18), to a CSV, or to a JSON file:

```python
import json
with open("voltaire-after-1750.json", "w", encoding="utf-8") as f:
    json.dump(matches, f, indent=2, default=str)
```

The `default=str` is a useful trick â€” it tells `json` to call `str()` on anything it doesn't natively know how to serialize, which catches `datetime` objects from Excel automatically.

## Running the code

This lesson uses `openpyxl`. Add it to your project once:

```bash
uv add openpyxl
```

Then save the snippet to a file â€” say `search_excel.py` â€” alongside `letters.xlsx` and run it:

```bash
uv run search_excel.py
```

`uv run` uses the project's Python and dependencies automatically; no virtualenv to activate. If you haven't set the project up yet, [Lesson 01](/course/lesson-01-introduction-to-python-for-dh/) walks through it.
