So far we've **read** spreadsheets. This lesson shows how to **write** them. Two libraries handle this well:

- **`xlsxwriter`** (used in the video) — write-only, very fast, supports every Excel feature (formats, charts, conditional formatting). Good when you're producing a polished output file.
- **`openpyxl`** — read *and* write. The right choice when you need to load an existing workbook and modify it.

We'll cover both.

## Installing

```bash
uv add xlsxwriter
uv add openpyxl
```

You only need both if you actually use both. For a brand-new file, either one works.

## Hello, workbook (xlsxwriter)

```python
import xlsxwriter

workbook = xlsxwriter.Workbook("output.xlsx")
worksheet = workbook.add_worksheet("Results")

worksheet.write(0, 0, "Hello, spreadsheet!")
workbook.close()
```

Cells are addressed `(row, column)`, both **0-indexed**. So `(0, 0)` is `A1`. (Note: openpyxl uses 1-indexed addresses. xlsxwriter is 0-indexed. Watch for this.)

## Writing rows of data

```python
import xlsxwriter

records = [
    {"sender": "Voltaire", "recipient": "Émilie",   "year": 1735},
    {"sender": "Voltaire", "recipient": "Diderot",  "year": 1759},
    {"sender": "Émilie",   "recipient": "Voltaire", "year": 1740},
]

workbook = xlsxwriter.Workbook("letters.xlsx")
worksheet = workbook.add_worksheet("Letters")

# 1. Header
header = ["sender", "recipient", "year"]
for c, label in enumerate(header):
    worksheet.write(0, c, label)

# 2. Rows
for r, record in enumerate(records, start=1):
    for c, key in enumerate(header):
        worksheet.write(r, c, record[key])

workbook.close()
```

`xlsxwriter` is smart about types: integers and floats are written as numbers, strings as strings. Lists, tuples, and other complex types it doesn't understand will raise an error — convert them yourself first.

## Adding a header format

Formats give Excel cells the visual treatment that makes a generated file feel finished:

```python
header_fmt = workbook.add_format({
    "bold": True,
    "bg_color": "#1f1f1f",
    "font_color": "#ffffff",
    "align": "center",
    "border": 1,
})

for c, label in enumerate(header):
    worksheet.write(0, c, label, header_fmt)
```

A few of the format options you'll reach for:

| Option | Effect |
| --- | --- |
| `bold`, `italic` | bold or italic text |
| `font_size`, `font_color` | typography |
| `bg_color` | cell fill |
| `align`, `valign` | horizontal and vertical alignment |
| `border`, `border_color` | cell borders |
| `num_format` | number/date formatting (e.g. `"#,##0"`, `"yyyy-mm-dd"`) |
| `text_wrap` | wrap long text inside the cell |

## Setting column widths and freezing the header

Two small touches that go a long way:

```python
worksheet.set_column(0, 0, 15)   # column A: 15 chars
worksheet.set_column(1, 1, 15)
worksheet.set_column(2, 2, 8)
worksheet.freeze_panes(1, 0)     # keep the first row visible while scrolling
```

## Writing dates correctly

Excel stores dates as a special number with date formatting on top. To write a Python `datetime` and have Excel display it as a date, use a date format:

```python
from datetime import date

date_fmt = workbook.add_format({"num_format": "yyyy-mm-dd"})
worksheet.write_datetime(1, 2, date(1735, 6, 1), date_fmt)
```

Without the format, the cell will show as a serial number like `42835`.

## Don't forget to close

`xlsxwriter` writes nothing to disk until you call `workbook.close()`. Forgetting this is the most common reason a script appears to "do nothing":

```python
workbook.close()
```

For the safer pattern, wrap your work in a `with` block — `xlsxwriter` supports it:

```python
with xlsxwriter.Workbook("output.xlsx") as workbook:
    worksheet = workbook.add_worksheet("Results")
    worksheet.write(0, 0, "Hello")
# closes automatically here
```

## The same task with openpyxl

If you want to **edit an existing** workbook — preserve its other sheets, add new rows, save it back — `openpyxl` is the tool. The shape is similar:

```python
from openpyxl import Workbook

wb = Workbook()
ws = wb.active
ws.title = "Letters"

ws.append(["sender", "recipient", "year"])
for record in records:
    ws.append([record["sender"], record["recipient"], record["year"]])

wb.save("letters.xlsx")
```

`openpyxl` is **1-indexed**. `ws.append(row)` adds a row at the bottom. `wb.save(...)` is the equivalent of `workbook.close()` in xlsxwriter.

## What if I just have a list of dicts?

For simple "write this data to a spreadsheet" needs, both pandas and polars do it in one line:

```python
import polars as pl
pl.DataFrame(records).write_excel("letters.xlsx")

# or:
import pandas as pd
pd.DataFrame(records).to_excel("letters.xlsx", index=False)
```

That trades the fine-grained control of formats for brevity. Useful when you just need a CSV-ish dump in Excel form.
