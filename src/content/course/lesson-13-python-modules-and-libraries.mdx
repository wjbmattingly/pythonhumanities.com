Most of the interesting work in a digital humanities project happens not in the code you write yourself but in the **libraries** you pull in: regex, `requests`, BeautifulSoup, spaCy, pandas, and so on. Using these is not plagiarism and it is not cheating. They are open source, free to use, and the entire ecosystem expects you to use them.

## What is a module?

A **module** (also called a **package** or **library**) is a block of code written by someone else — sometimes a single developer, sometimes a research lab at MIT, Stanford, or an institute like AllenAI. Modules let you bring complex behavior into your script with a single import. Most modules have good documentation on their main website; if you Google the module name, you'll usually land on it within the first two results.

## Installing a module

In modern Python (and throughout this course), we install dependencies with [`uv`](/course/lesson-01-introduction-to-python-for-dh/). Inside your project folder:

```bash
uv add <module-name>
```

For example, to install the regex library we're about to use:

```bash
uv add regex
```

(Python ships with a built-in regex module called `re`, so for `re` you don't actually need `uv add`. But for almost any *third-party* library — `requests`, `beautifulsoup4`, `spacy`, `pandas` — you do.)

You may see older tutorials use `pip install <module>`. That still works, but `uv add` writes the dependency into your `pyproject.toml` so the project is reproducible.

## Importing a module

Once installed, bring the module into your script with `import`:

```python
import re
```

Most coders put all their imports at the top of a script. (Machine-learning notebooks sometimes scatter imports next to where they're used, but for the purposes of this series, keep them at the top.)

You can also rename a module on import — common for libraries with long names:

```python
import pandas as pd
import numpy as np
```

Or import just one piece of a module:

```python
from bs4 import BeautifulSoup
```
