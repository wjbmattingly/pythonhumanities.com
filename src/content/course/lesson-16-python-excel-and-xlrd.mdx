A great deal of humanities data lives in spreadsheets. This lesson covers how to read one from Python — open a workbook, pick a sheet, walk the cells. The video uses `xlrd`; the version below uses `openpyxl`, which is the right tool for modern `.xlsx` files. I'll cover both.

## A note on file formats

There are two Excel formats you'll encounter:

- **`.xlsx`** — the modern format (Excel 2007 and later). Use **`openpyxl`** to read and write it. This is what you'll use 95% of the time.
- **`.xls`** — the legacy format (Excel 97–2003). Use **`xlrd`** to read it. (Modern `xlrd` only reads, and only `.xls`. Writing `.xls` is essentially obsolete.)

If you have a choice, save as `.xlsx`. If you're inheriting old data, fine — read it with `xlrd`, save it as `.xlsx` with `openpyxl`, and use the modern format from there.

There's also **pandas** (`pd.read_excel("data.xlsx")`) and **polars** (`pl.read_excel("data.xlsx")`), which both load a spreadsheet directly into a tabular data frame. For straight data analysis these are wonderful. For learning what the data actually contains and walking it cell by cell, `openpyxl` is more direct, which is why we start there.

## Reading an `.xlsx` workbook with openpyxl

```python
from openpyxl import load_workbook

wb = load_workbook("data.xlsx", data_only=True)
print(wb.sheetnames)              # ['Sheet1', 'Letters', 'Notes']

sheet = wb["Letters"]             # by name
sheet = wb.active                 # or the currently active sheet
sheet = wb.worksheets[0]          # or by index
print(sheet)
```

`data_only=True` is worth knowing about. By default, `openpyxl` returns **formulas** for cells that contain them (e.g. `=SUM(A1:A10)`). With `data_only=True`, it returns the **last computed value** Excel saved. For research data you almost always want the values, not the formulas.

## Reading individual cells

Two ways:

```python
from openpyxl import load_workbook

wb = load_workbook("data.xlsx", data_only=True)
sheet = wb.active

print(sheet["A1"].value)           # by Excel-style address
print(sheet.cell(row=1, column=1).value)   # by 1-based row/column numbers
```

Note: `openpyxl` is **1-indexed** — the first row is row `1`, not row `0`. (xlrd, which we'll see in a moment, is 0-indexed. Watch for this.)

Sheet dimensions:

```python
from openpyxl import load_workbook

wb = load_workbook("data.xlsx", data_only=True)
sheet = wb.active

print(sheet.max_row)
print(sheet.max_column)
```

## Walking every row

The most natural pattern — `iter_rows` yields each row as a tuple of cells:

```python
from openpyxl import load_workbook

wb = load_workbook("data.xlsx", data_only=True)
sheet = wb.active

for row in sheet.iter_rows(min_row=2, values_only=True):
    sender, recipient, year = row
    print(sender, recipient, year)
```

Three things to know:

- `min_row=2` skips the header row.
- `values_only=True` gives you the raw values (strings, numbers, dates) instead of cell objects. Almost always what you want.
- The `for` unpacking (`sender, recipient, year = row`) only works when every row has the same number of columns. For variable-width sheets, capture the whole tuple as `row` and index into it.

## Header row → list of dictionaries

The most useful shape for further work — turn each row into a dictionary keyed by the header:

```python
from openpyxl import load_workbook

wb = load_workbook("data.xlsx", data_only=True)
sheet = wb.active

rows = list(sheet.iter_rows(values_only=True))
header, *body = rows

records = [dict(zip(header, r)) for r in body]
print(records[0])
# {'sender': 'Voltaire', 'recipient': 'Émilie', 'year': 1735}
```

From here every later operation — search, sort, filter, count — is just a list-of-dicts operation, which we covered in Lesson 07.

## Reading a legacy `.xls` workbook with xlrd

For files that are still in the old format:

```python
import xlrd

book = xlrd.open_workbook("data.xls")
print(book.sheet_names())

sheet = book.sheet_by_index(0)
print(sheet.nrows, sheet.ncols)

for r in range(sheet.nrows):
    for c in range(sheet.ncols):
        print(r, c, sheet.cell_value(r, c))
```

xlrd is **0-indexed** — the first row is row `0`, the first column is column `0`. The mental model is the same as openpyxl, but the offsets are different.

## Don't forget dates

Dates in Excel are stored as a number (days since 1899/12/30, more or less). Both libraries can decode them, but the details differ:

- **openpyxl** with `data_only=True` and `values_only=True` gives you Python `datetime.datetime` objects directly.
- **xlrd** gives you the raw number; convert with `xlrd.xldate.xldate_as_datetime(value, book.datemode)`.

If a year is showing up as `42835` instead of `2017`, you're looking at a serial date that hasn't been converted.

## Running the code

This lesson uses `openpyxl` for `.xlsx` files and `xlrd` for legacy `.xls` files. Add them to your project once:

```bash
uv add openpyxl xlrd
```

Then save the snippet to a file — say `read_excel.py` — alongside your spreadsheet and run it:

```bash
uv run read_excel.py
```

`uv run` uses the project's Python and dependencies automatically; no virtualenv to activate. If you haven't set the project up yet, [Lesson 01](/course/lesson-01-introduction-to-python-for-dh/) walks through it.
