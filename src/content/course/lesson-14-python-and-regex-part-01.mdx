The first major library we'll work with in this series is **Regex** (regular expressions). Digital humanists work heavily with text, and one of the most powerful tools for handling strings is regex. It allows us to interact with strings in far more dynamic ways than the built-in Python string methods do — in particular, we can account for *consistent variance* in a text.

In this lesson, we work through a real-world problem: imagine a dataset where dates are inconsistently formatted. Sometimes the month has a leading zero (`02`), sometimes it does not (`2`). Sometimes the year is two digits, sometimes four. Using only string methods, handling all of those variants would take a lot of code. With regex, we can do it in a single line.

Regex is implemented in everything from text editors to web frameworks because the same pattern language describes "find me dates that look like this" no matter where it runs.

## Importing regex

Python's regex library is built in and called `re`. As we learned in [Lesson 13](/course/lesson-13-python-modules-and-libraries/), we bring it in with an `import` at the top of the script:

```python
import re
```

## Finding all digits

Here are two strings to play with:

```python
import re

new_string = "I have 1 cat, 2 dogs, and 3 birds. There are 5 of each. So 5 + 5 + 5 = 15."
description = "The event happened on 5/3/1989 and again on 11/12/95."
```

Let's start with the simplest pattern: find every digit in `new_string`.

```python
print(re.findall(r"\d", new_string))
```

`re.findall(pattern, string)` returns a list of every match. The `r` before `"\d"` makes it a **raw string**, which means Python won't try to interpret the backslash itself — it hands the pattern to regex untouched. Inside the pattern, `\d` is regex shorthand for "any single digit."

Output:

```
['1', '2', '3', '5', '5', '5', '1', '5']
```

Notice that `15` came back as `'1'` and `'5'` separately. `\d` only matches a single digit at a time and treats each one as its own match.

## Matching one *or* two digits

Replace `\d` with `[0-9]{1,2}` and run it again:

```python
print(re.findall(r"[0-9]{1,2}", new_string))
```

Output:

```
['1', '2', '3', '5', '5', '5', '15']
```

`[0-9]` is a **character class** — it matches any digit from `0` through `9`. The braces `{1,2}` are a **quantifier** meaning "match between 1 and 2 of the preceding thing." Now `15` is found as a single two-digit match.

## Matching whole dates

Now use a richer pattern against `description` to pull out anything that looks like a date:

```python
print(re.findall(r"[0-9]{1,2}/[0-9]{1,2}/[0-9]{2,4}", description))
```

Read the pattern left to right:

- `[0-9]{1,2}` — a 1- or 2-digit number (the month).
- `/` — a literal slash.
- `[0-9]{1,2}` — another 1- or 2-digit number (the day).
- `/` — another slash.
- `[0-9]{2,4}` — a 2- to 4-digit number (the year).

Output:

```
['5/3/1989', '11/12/95']
```

Both dates were captured even though they have different shapes — one with single-digit month/day and a four-digit year, the other with two-digit everything. That tolerance for variation is exactly what regex is good for.
