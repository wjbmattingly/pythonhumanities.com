Classes are the mark of sophisticated code. They are something you do not have to use, but they make your code much cleaner because they reduce duplicated logic, much like functions do. A **class** is a template that bundles together some data (the things it knows) and some functions (the things it can do).

Throughout this lesson we'll work with the same example used in the video: a `Emperor` class for representing rulers from history.

## Defining a class

We define a class with the `class` keyword followed by the class name. Then, just like functions, we use a colon and indent the body:

```python
class Emperor:
    def __init__(self, name, birth, coron, death):
        self.name = name
        self.birth = birth
        self.coron = coron
        self.death = death
```

The first method, `__init__`, is special. It is the **initializer** — it runs whenever you create a new `Emperor`. The first argument is always `self`, which refers to the instance being built. The remaining arguments are whatever you want to capture about each emperor: name, birth year, coronation year, year of death.

Inside `__init__`, we attach those arguments to `self` so that they stick around as attributes of the object.

## Creating an instance

To make a specific emperor, call the class as if it were a function:

```python
p1 = Emperor("Charlemagne", 742, 800, 814)
```

`p1` is now an **instance** of `Emperor`. If you print it directly, you'll see something like:

```
<__main__.Emperor object at 0x7f9b49a4ae80>
```

That looks like gibberish to us, but Python is just telling us "this is an Emperor object stored at this memory address." To inspect the actual data, convert it to a dictionary:

```python
print(p1.__dict__)
```

Or, more idiomatic since Python 3.5:

```python
print(vars(p1))
```

Both yield the same result — a dictionary of the attributes you stored.

## Accessing a single attribute

You don't always need the whole dictionary. You can ask for any one attribute by name:

```python
print(p1.birth)
```

This returns `742`.

## Adding methods

The real power of a class is that it can carry **methods** — functions that operate on `self`. Here we add three: one to compute lifespan, one to render a short bio, and one to compare two emperors.

```python
class Emperor:
    def __init__(self, name, birth, coron, death):
        self.name = name
        self.birth = birth
        self.coron = coron
        self.death = death

    def lifespan(self):
        return self.death - self.birth

    def prose(self):
        return f"{self.name} was born in {self.birth} and died in {self.death}."

    def older_than(self, other):
        return self.lifespan() > other.lifespan()
```

Calling them looks like calling a function on the object:

```python
p1 = Emperor("Charlemagne", 742, 800, 814)
p2 = Emperor("Theuderic IV", 712, 721, 737)

print(p1.lifespan())
print(p1.prose())
print(p1.older_than(p2))
```

## Why bother with classes?

Why not just use a dictionary for each emperor? Two reasons.

First, classes make it dramatically easier to create many similar objects. One `class` definition powers an unlimited number of instances — each instance is just one line.

Second, classes carry behavior with their data. A dictionary is inert; a class can have methods that know how to operate on the things it stores. As your scripts get bigger, that bundling pays off.
